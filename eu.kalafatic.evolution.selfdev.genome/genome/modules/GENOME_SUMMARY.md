# AI SUMMARY: Genome System

## What is this subsystem?
The persistent memory of EVO. It stores "Evolutionary Lessons," patterns, and traits.

## Why does it exist?
To prevent architectural "amnesia" between AI sessions. Future sessions read the Genome to understand established patterns without re-analyzing the entire codebase.

## Key Components
- **Genome A**: Optimized Instructional Prompts.
- **Genome B**: Curated context and metadata.
- **Architectural Genes**: Reusable patterns (e.g., "Service-Repository").

## Interaction Map
- Populated by **Darwin Engine** milestones.
- Managed by **LocalGenomeRepository**.
- Used by **SecondhandUpgradeEngine** for pattern migration.
