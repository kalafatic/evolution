# CONTROL SUMMARY

* **Iteration Owner:** IterationManager (Coordinates the Darwin lifecycle and phases)
* **Variant Creation Owner:** DarwinEngine (Agent-driven proposal generation)
* **Selection Owner:** IterationManager (Deterministic scoring via `calculateScore`)
* **Mutation Owner:** GitManager (Triggered by IterationManager to apply the selected variant)

## Conflict Points

* **[013, 014]**: The Orchestrator re-runs intent classification and analysis on tasks that were already generated by the `TaskPlanner` as part of a structured Darwin strategy.
* **[017]**: `executeTaskWithRetries` asks the agent for a "tactical plan" for a task that was already defined with high specificity by the `DarwinEngine` and `TaskPlanner`.
* **[031]**: The final `evaluate()` call is redundant as the `bestVariant` was already fully evaluated during the parallel `EVALUATE` phase; only a sanity check is achieved.

## Minimal Call Chain

```
IterationManager.runDarwin()
→ DarwinEngine.generateVariants()
→ IterationManager.evaluateVariantsInternal()
  → IterationManager.evaluateVariantParallel()
    → TaskExecutor.executeTasks()
      → EvolutionOrchestrator.handle()
        → EvolutionOrchestrator.executeTaskWithRetries() (PEV Loop)
→ IterationManager (Select Best)
→ GitManager.merge()
→ GitManager.commit()
```

## Variant Tracking

* **Creation**: 2-3 `BranchVariant` objects created in `DarwinEngine.generateVariants()`.
* **Storage**: Stored in a `List<BranchVariant>` within `IterationManager.runDarwin()`.
* **Passage**: Passed to `evaluateVariantsInternal()`, then individually to `evaluateVariantParallel()`.
* **Lost/Overwritten**: Non-selected variants are deleted (branches/worktrees) at the end of `runDarwin()`, but their metadata is persisted to `memoryService` for future trajectory analysis.
