# EVO Presentation: Step-by-Step AI Evolution Platform Showcase

This document outlines the presentation flow for capturing a video demonstration of the EVO platform. Each step corresponds to a video segment and includes subtitles for AI processing.

---

## 0. Introduction: Motivation & Vision
**Action**: Display a high-level overview or slide with the WHY, WHAT, and HOW.

### WHY? (The Motivation)
**Subtitles**:
- Why build EVO? The motivation is three-fold: Personal, Economical, and Political.
- **Personal**: It’s about the joy of creation, staying at the absolute frontier of technology, and ensuring long-term job relevance in an AI-driven industry.
- **Economical**: For both users and developers, EVO builds "priceless know-how." It transforms the user from a consumer into a master of the AI lifecycle.
- **Political**: In an era of increasing censorship and the global "GPU/LLM war," EVO provides independence. It’s about local control and resisting centralized AI authority.

### WHAT? (The Scope)
**Subtitles**:
- EVO is a comprehensive platform for the entire AI lifecycle.
- It integrates local LLMs via Ollama, the Model Context Protocol (MCP) for ecosystem connectivity, and a deterministic evolutionary Orchestrator.
- The ultimate vision is **local self-development**: an AI platform that can autonomously improve its own source code and architecture on your local machine.

### HOW? (The Foundation)
**Subtitles**:
- Built on a rock-solid foundation of open technologies: **Java, Eclipse RCP, and the Eclipse Modeling Framework (EMF)**.
- We leverage Ollama for local execution and frontier tools like **Grok, ChatGPT, and DeepSeek** for advanced reasoning.
- My role is the **Architect** – using agents like **Jules** to generate code while I maintain the structural integrity and evolutionary direction.

**Subtitles**:

- **Page 2: Config Details** – Setting the default target path and orchestration filename.
- **Page 3: Git Settings** – Configuring repository URLs for both the Evolution logic and the Workspace source.
- **Page 4: Ollama Settings** – Connecting to the local LLM instance for autonomous enhancements.
- **Page 5: LLM Settings** – Selecting the primary frontier model (e.g., GPT-4o) and temperature.
- **Page 6: Maven Settings** – Defining build goals and profiles for the automated execution loop.
- **Page 7: AiChat Settings** – Customizing the chat interface and proxy configurations.
- **Page 8: Neuron AI Settings** – Mapping computational units for architectural modeling.
- **Page 9: Agent Settings** – Selecting specialized agents like Analytic, Critic, and Architect.
- **Page 10: Supervisor Settings** – Configuring the decision authority and safety boundaries.

## 3. UI Discovery: Multi-page Editor Walkthrough
**Action**: Switch to the "Evo" perspective. Open an `.evo` file and click through every tab in the Multi-page Editor in sequence.

**Subtitles**:
- The **Multi-page Editor** is the nerve center of an Evolution session. Let's explore every tab in the order they appear.
- **1. Forge Models** – The palette for dragging and dropping computational units to design the project's AI architecture.
- **2. AI Chat** – The primary communication channel. Engage in Darwinian dialogues and trigger evolutionary tasks directly.
- **3. Architecture** – Visualizing the structural genome. Discover subsystems, use cases, and relationship maps.
- **4. Development** – The pre-flight dashboard. It tracks Git status, Maven goals, and LLM health before starting the evolution loop.
- **5. Iterations** – Visualizing the Darwinian tree. Here we see Intents, Dimensions, and the branching lineage of successful variants.
- **6. Task Stack** – A real-time monitor for the execution queue. Every atomic task, from analysis to code generation, is tracked here.
- **7. Server** – Managing the local Evolution Server lifecycle and port configurations for UI synchronization.
- **8. MCP Settings** – Configuring the Model Context Protocol. Connect tools, resources, and prompts from external ecosystems.
- **9. Browser** – An integrated web environment for documentation research and AI-assisted browsing.
- **10. Tests** – Monitoring test suites and coverage metrics. Evolution success is often tied to these validation signals.
- **11. Context** – Inspecting the "Active Genome". See exactly which files and metadata are currently being fed to the AI.
- **12. Approval** – The human-in-the-loop checkpoint. Review proposed changes and decide which evolutionary branch to activate.
- **13. Peer Review** – Managing collaborative feedback. Review comments and decisions from both human and AI peers.
- **14. Graph** – A high-level visual map of the entire task workflow and its current state.
- **15. Tools** – Quick access to Git automation, Maven builds, and environment setup utilities.
- **16. Properties** – Deep-dive into the configuration metadata of the current orchestrator.
- **17. Settings** – Global preferences and model-specific tuning parameters for the session.
- **18. Compare** – Visualizing the delta between the proposed evolution and the current Git HEAD.
- **19. Editor** – The raw XML/EMF view of the underlying orchestration model for direct manual adjustment.
