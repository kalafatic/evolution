* `DARWIN_ITERATION.md`: Traditional agents follow a linear path. The AI Evolution Platform uses **Darwinian Evolution** to explore the "Intent Space" through parallel branching. 1. **Mutation**: `DarwinEngine` generates multiple proposal branches (`BranchVariant`) for a given goal. 2. **Exploration**: Each branch is executed in an isolated worktree. 3. **Evaluation**: Evaluators produce signals for each branch. 4. **Selection**: `DecisionResolver` selects a winner based on aggregated signals. 5. **Consolidation**: The winner is merged into the main branch, and the cycle repeats. EPS is a metric that determines the complexity of the orchestration flow. - **Formula**: `min(1.0, (Ambiguity * 0.4) + (Risk * 0.4) + (Complexity * 0.2))` - **EPS < 0.25**: Atomic Flow (Direct execution). - **0.25 <= EPS < 0.6**: Iterative Flow (Light refinement). - **EPS >= 0.6**: Full Darwin Flow (Evolutionary branching). Every variant maintains a lineage, allowing the kernel to trace the "ancestry" of a solution back to its original hypothesis. This is critical for both observability and the ability to roll back to a stable genetic state. Darwin variants are not random. They are grounded in `IntentHypothesis` objects generated by the `IntentExpansionEngine`, ensuring that exploration is targeted and engineering-focused.
