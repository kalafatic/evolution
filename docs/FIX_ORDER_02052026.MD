# Implementation Roadmap: Hardened Evolutionary Kernel (Final Synthesis)

## Executive Summary
The system is now defined as a **Deterministic Evolutionary OS Kernel**. Fixing the **Self-Dev + Iterative + Darwin** scenario is the absolute priority as it establishes the production-grade control plane for all other system projections.

---

## Hardened Implementation Strategy (Synthesis)

### PHASE 1: Control Plane & State Enforcement (Model)
- **SystemState (Hardened):** INIT, ANALYZING, PLAN_LOCKED, EXECUTING, VERIFYING, MUTATING, DONE, FAILED, RECOVERING.
- **TransitionToken & SystemStateHolder:** Make state immutable outside `IterationManager`. Use a `TransitionToken` pattern (package-private/final) required by `applyTransition()` to prevent state bypass via reflection, hacks, or external components.
- **Event Sourcing Persistence:** Store the full iteration history as an immutable log of events (transitions, plans, execution results, evaluations). This enables "Time-Travel Debugging" and evolution visualization.

### PHASE 2: Sole Transition Authority (IterationManager)
- **Transition Skeleton (MANDATORY):**
  ```java
  void transition(SystemState to, Context ctx) {
      TransitionToken token = createToken(); // internal to IterationManager
      validateTransition(current, to);
      validateEntryConditions(to, ctx);
      terminationGuard.check();               // triggers on max_iter, budget, or stagnation

      persistence.snapshotLight();            // lightweight metadata always
      stateHolder.applyTransition(token, to);

      executeExitActions();
      executeEntryActions(to, ctx);           // e.g., start execution, trigger verifier
      persistence.snapshotIfHeavy(to);        // full plan/result data on major gates

      logTransition(current, to, ctx);
  }
  ```
- **Pluggable ProgressPolicy:** Implement stagnation detection as a `boolean isStagnant(IterationHistory, DarwinMetrics)` check.

### PHASE 3: Deterministic Verification & Intelligence
- **Pure Verifier:** Refactor the `VERIFYING` phase into a pure, versioned function: `EvaluationResult verify(ExecutionResult, FinalPlan)`. The `EvaluationResult` must include the `VerifierVersion` for reproducibility.
- **Intelligence plane isolation:** `DarwinEngine` generates and evaluates candidates but is strictly prohibited from selection or transition decisions.

### PHASE 4: Stateless Execution & Gating (Orchestrator)
- **Blind Executor:** Strip all interpretation and planning autonomy.
- **Strict Gating:** `if (state != EXECUTING || !hasFinalPlan) fail();`.

---

## Final Architectural Conclusion
The system architecture has successfully converged into a **closed-loop state transition machine with externalized intelligence modules**.
1. **Control Plane (Kernel):** `IterationManager` + State Machine.
2. **Intelligence Plane:** `DarwinEngine` + `Verifier` (Replaceable).
3. **Execution Plane:** `EvolutionOrchestrator` (Replaceable).

This structure is resilient to flaky LLMs, production operational surprises, and interpretive drift. **Verdict: Ship this.**
