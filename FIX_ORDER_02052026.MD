# Implementation Roadmap: Hardened Evolutionary Kernel (Final Hardening)

## Executive Summary
Fixing the **Self-Dev + Iterative + Darwin** scenario builds the definitive architectural superset. The system transitions from a "philosophical" state machine to a production-grade **Evolutionary Kernel** where the control, intelligence, and execution planes are strictly decoupled and failure-resistant.

---

## Hardened Implementation Strategy

### PHASE 1: Control Plane & Entry Invariants (Model)
**Target:** `eu.kalafatic.evolution.model`
Define the state space and the hard entry requirements for each node.
- **SystemState:** INIT, ANALYZING, PLAN_LOCKED, EXECUTING, VERIFYING, DONE, MUTATING.
- **STATE ENTRY RULES (MANDATORY):**
  - **ANALYZING:** input exists.
  - **PLAN_LOCKED:** finalPlan != null.
  - **EXECUTING:** finalPlan != null.
  - **VERIFYING:** executionResult != null.
  - **MUTATING:** evaluationResult != null.
- **Transition Table Enforcement:** Implement `isValidTransition(from, to)` based on the hardened DAG.

### PHASE 2: Sole Transition Authority (IterationManager)
**Target:** `IterationManager.java`
Centralize converting intelligence data into control decisions.
- **Verifying Contract:** IterationManager must consume `executionResult` and produce a deterministic `evaluationResult` (success/failure + score).
- **Control Decision:** ONLY IterationManager decides the next transition:
  - success → DONE.
  - failure → MUTATING.
- **Mutation Isolation:** The `MUTATING` state produces new plan candidates from the `evaluationResult` but is strictly prohibited from execution.

### PHASE 3: Stateless Execution Plane (Orchestrator)
**Target:** `EvolutionOrchestrator.java`, `TaskExecutor.java`
- **Execution Gating:** `if (state != EXECUTING || !iteration.hasFinalPlan()) fail();`.
- **Blind Executor:** Strip all interpretation, planning, or evaluation privileges. The Orchestrator is purely reactive to the `EXECUTING` state.

### PHASE 4: Mode Projections (Convergence)
**Target:** `AiChatPage.java`, `SelfDevSupervisor.java`
- **Validated Projections:**
  - **Simple Chat:** INIT → DONE.
  - **Standard Iteration:** INIT → ANALYZING → PLAN_LOCKED → EXECUTING → DONE.
  - **Darwin:** INIT → ANALYZING → PLAN_LOCKED → EXECUTING → VERIFYING → MUTATING → loop.

---

## Final Architectural Conclusion
The system is now defined by three distinct planes:
1. **Control Plane (State Machine):** Owns transitions and enforcement rules.
2. **Intelligence Plane (DarwinEngine/Planner):** External module providing intelligence as a service (raw data).
3. **Execution Plane (Orchestrator):** Stateless engine executing immutable plans.

This separation ensures that **only one component (IterationManager)** converts data into decisions, eliminating the interpretive drift and triple-planning cycles identified in the original audit.
