# Implementation Roadmap: Hardened Evolutionary Kernel (Refined)

## Executive Summary
Fixing the **Self-Dev + Iterative + Darwin** scenario remains the priority as it builds the architectural superset. However, the architecture must transition from being "philosophically correct" to "execution-proof" by enforcing a strict state machine projection model.

---

## Hardened Implementation Strategy

### PHASE 1: Evolutionary Kernel & Transition Table (Model)
**Target:** `eu.kalafatic.evolution.model`
Define the state space and the valid paths through it.
- **SystemState:** INIT, ANALYZING, PLAN_LOCKED, EXECUTING, VERIFYING, DONE.
- **Valid Transition Table (MANDATORY):**
  - INIT → ANALYZING
  - ANALYZING → PLAN_LOCKED
  - PLAN_LOCKED → EXECUTING
  - EXECUTING → VERIFYING
  - VERIFYING → DONE | MUTATING
  - MUTATING → PLAN_LOCKED
- **Immutable Plan:** Add `finalPlan` field to `Task` and `Iteration`.

### PHASE 2: State Transition Authority (IterationManager)
**Target:** `IterationManager.java`
Centralize all control logic and enforce transition rules.
- **Enforcement:** Implement `transition(current, next)` with `validateTransition()` logic. Throw `IllegalStateException` on illegal jumps.
- **Decision Finality:** IterationManager is the ONLY place where state changes occur.
- **Transition Logging:** Mandatory logging for every state shift: `[STATE] FROM → TO (Reason)`.

### PHASE 3: Stateless Execution & Gating (Orchestrator)
**Target:** `EvolutionOrchestrator.java`, `TaskExecutor.java`
Remove tactical autonomy and implement strict gating.
- **Execution Gate:** Before the Orchestrator runs: `if (state != EXECUTING || !hasFinalPlan) return/fail;`.
- **Simplification:** Strip all intent classification, re-analysis, and tactical planning logic from the Orchestrator. It becomes a blind executor.

### PHASE 4: Mode Projections (Convergence)
**Target:** `AiChatPage.java`, `SelfDevSupervisor.java`
Unify all entry points into the single state machine.
- **Mode Projections:**
  - **Simple Chat:** INIT → DONE
  - **Standard Iteration:** INIT → ANALYZING → PLAN_LOCKED → EXECUTING → DONE
  - **Darwin:** INIT → ANALYZING → PLAN_LOCKED → EXECUTING → VERIFYING → MUTATING → loop
- Refactor all triggers to start the `IterationManager` engine.

---

## Final Architectural Conclusion
The system is no longer a multi-agent orchestration system; it is a **deterministic state-transition-driven evolutionary kernel**. By treating every operational mode as a specific projection of the same underlying state machine, we ensure that behavior is enforced structurally, eliminating the "interpretation creep" that caused planning drift in earlier versions.
