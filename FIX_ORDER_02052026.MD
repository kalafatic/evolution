# Implementation Roadmap: Hardened Evolutionary Kernel (Production Grade)

## Executive Summary
Fixing the **Self-Dev + Iterative + Darwin** scenario builds the definitive architectural superset. The system is no longer a multi-agent system; it is a **Deterministic Evolutionary OS Kernel**.

---

## Hardened Implementation Strategy

### PHASE 1: Control Plane & State Enforcement (Model)
**Target:** `eu.kalafatic.evolution.model`
- **SystemState:** INIT, ANALYZING, PLAN_LOCKED, EXECUTING, VERIFYING, MUTATING, DONE, FAILED, RECOVERING.
- **TransitionToken Pattern:** Make state immutable outside `IterationManager`. Use a `TransitionToken` required by `applyTransition()` to prevent bypass via reflection or test hacks.
- **Event Sourcing:** Store full iteration history (transitions, plans, execution results, evaluations) as a sequence of immutable events for replay and debugging.

### PHASE 2: Sole Transition Authority (IterationManager)
**Target:** `IterationManager.java`
- **Single-Writer Constraint:** Enforce a single-threaded transition queue to prevent race conditions in state shifts.
- **State Entry Invariants:**
  - **ANALYZING:** input exists.
  - **PLAN_LOCKED:** finalPlan != null.
  - **VERIFYING:** executionResult exists; uses a **Pure Deterministic Verifier** (ExecutionResult + Plan -> EvaluationResult).
  - **RECOVERING:** must rebuild context and optionally rollback to last stable snapshot.
- **Termination Guards:** Mandatory STOP if max_iterations reached, time_budget exceeded, or score stagnation detected.

### PHASE 3: Stateless Execution Plane (Orchestrator)
**Target:** `EvolutionOrchestrator.java`
- **Blind Executor:** Strip all interpretation and planning autonomy.
- **Strict Gating:** `if (state != EXECUTING || !hasFinalPlan) fail();`.

### PHASE 4: Intelligence Plane Isolation (DarwinEngine)
**Target:** `DarwinEngine.java`
- **Candidate Generation:** Strictly responsible for generating and evaluating candidates.
- **Control Decoupling:** DarwinEngine MUST NOT select the best candidate or influence the state transition. It returns raw data to the `IterationManager`.

---

## Final Architectural Conclusion
The system is now a **closed-loop state transition machine with externalized intelligence modules**. By centralizing all decision logic in the `IterationManager` and enforcing state invariants through a `TransitionToken` pattern, we eliminate interpretive drift and provide the failure-resistance required for stable autonomous development.
